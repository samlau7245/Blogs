# 类目

# 属性

属性关联？

商品属性指的是商品的特征信息。型号、颜色、尺码、容量等等都是属性。

  • 不同的商品拥有部分相同的属性，例如标题、品牌等等。
  • 不同的商品有各自独有的属性，如手机有内存，鞋子有尺码等等。
  • 有部分的属性参与了商品的价格销售，如手机的内存中，不同大小的内存价格也是不一样的。

# 属性分类

根据属性的用途可以把属性分类为：

  • 基础属性：所有的商品都有的通用属性，如标题、品牌、品类等
  • 销售属性：影响商品销售价格的属性，不同类型的的商品销售各不相同，如手机的颜色、内存等
  • 搜索属性：参与到商品搜索中的属性，如标题、品牌、品类等w
  • 特有属性/商品属性：不同商品各自所独有的属性，如鞋子的使用人群、鞋面材质，手表的机芯等
  • 非关键属性：是除关键属性、销售属性外的其他属性，如手机的手机接口类型。非关键属性不一定是非必填项，不过有时为了商品信息完整，也会设为必填项。

# 属性关联类目

相同类目的商品它们所有拥有的特殊属性基本相同，针对于这个特性，我们通常将产品的特殊属性和品类绑定在一起。

当商品在基础属性中设置了对应的类目后，就可以关联获得已经绑定的所有特殊属性配置。

# 属性继承

属性继承每一个层级的商品属性继承上一层级的商品属性，将每个层级的属性提取出来，便组成了一个商品的完整的属性表达。

1级类目有属性 A，2级类目有属性 B，3级类目有属性C，那3级类日下的商品 SKU1 就具有属性 A、B、C。 比如实物商品佳能 EOS 800D（在类目“数码”一“摄影摄像”一，“单反相机”中）就有商品毛重、像素、套头三个属性需要填写。

# 表单维护

  • 文本框：维护文字性的描述，并且没有明显规律的内容，如：使用场景、工艺描述等
  • 单选框：维护属性值可以被穷举的属性，如性别：男、女；季节：春、夏、秋、冬等
  • 复选框：维护属性值可以同时被多选的属性，如爱好、标签等
  • 下拉框：维护需要单选或多选但是属性值又比较多的属性，如颜色、材质等
  • 时间日期: 维护需要精确时间的属性，如上市日期等
  • 自定义组件：除了上述常见的表单组件，还可以自行根据需要开发相应的组件

# 个性化管理

为了商户为了凸显各自的产品特征，通常会加入自定义的属性。

# 属性模版

平台根据商品所属类目来确定需要填写的属性，如将手机相关的属性：品牌、CPU品牌、电池容量、屏幕类型等等固定成手机类目的属性模版，这样商家只要选择手机这个类目，就能直接带出手机的属性模版，直接填写，提高效率。

疑问

  • Q：为什么需要属性库(池)管理？
    • A：不同的商品可能有相同的属性，为了避免重复创建属性，方便管理属性，可以建立一个属性库。

基础属性或者说规格参数，对应属性分组，也就是说一个属性分组里面可以包含多个基础属性。
