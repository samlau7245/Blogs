# 概述

  • SPU： Standard Product Unit(标准化产品单元)： 如 iPhone 6S就是一个SPU，无论是到哪个平台查询iPhone 6S，基本的商品属性都是一致的。
  • SKU：Stock Keeping Unit(库存量单元)：SPU + 销售属性。 如 iPhone 6S + 颜色、内存等等销售属性，就组成了一个SKU.

电商系统中，对于商品信息的维护，其本质就是对一堆属性和属性值的管理。

# 商品属性分析

所有的属性可分类为： 基础属性、销售属性。

# 基础属性

对于所有商品都包含的属性，都可以归纳到基础属性中，如商品名称、品牌、类目、状态等等。

  • 类目：由于商品的特有属性和搜索属性都关联在品类上，所以商品品类是必须选择的。
  • 状态: 上架/下架，确定是否在前台展示当前商品。这是整个商品的全局控制；如果上架，则之后关联的SKU也一同上架；反之则一样。
  • 商品类型：单品/复合商品;部分平台上支持打包出售的模式，也就是一个商品实际上是包含多个关联子商品的。可以通过商品类型字段先标记出当前商品是单品还是复合商品；如果是复合商品，之后还需要关联对应子商品。

问题：

  • 通过属性创建的商品怎么在表中展示？

# 特有属性

在商品信息维护时，当选定品类后，我们就能获得已经设置好的配置内容，根据属性配置好的内容进行维护。

# 销售属性

对于销售属性的维护时，注意：

  • 在完成销售属性值的维护后，需要更具属性组合来生成SKU。
  • 销售属性值的个性化设置：如果一款手机中的相同红色，不同的商户有不用的叫法，如炫彩红、玫瑰红等等。

提示

  • 销售属性在这里进行配置？？？
  • 销售属性不能和其他分类的属性混在一起。因为销售属性的参数值会根据用户的选择随时变动，其他商品属性是一份数据，不会变动。也就是SKU属性和SPU属性的区别。

当维护好销售属性和销售属性值后，就能通过组合生成商品唯一的SKU，在之后商品的价格、订单、库存等等属性信息，都需要与SKU直接挂钩。

# 吊牌价

供应商在商品出厂时，为商品所设置的一个市场销售参考价。价格通常会和商品的一些质检信息一同写在商品的吊牌上。

# 日常销售价

商品在没有参与活动时所设置的出售价格。

  • 对于个体户：根据进货价设定日常销售价。
  • 对于大平台：根据采购价和期望的利润来设定日常销售价。

# 活动销售价

商品参与活动时所设置的销售价格就是活动销售价，活动销售价只有在活动期间有效。当活动过期，商品售价又会使用日常销售价。

# 库存

对于实际可售库存，通常采用手动录入的方式；让商户自己维护SKU的销售数量。

# 上架/下架

在基础属性上设置商品的上架/下架操作，再通过在具体的SKU上设置上架/下架操作，可以更加细粒度的管理。

# 图片维护

商品的图片做展示的位置：

  • 列表图：主要是在搜索列表所展示的图片。
  • SKU图：为每个SKU上传对应的一组展示图片，主要用于商品详情页的展示上。
  • 默认图组：如果对应的SKU未设置展示图片，则显示默认的图组。
